Our Story — Critical Reflection

Summary of work completed: For this project we designed and implemented a six-page commercial website for "Beans World" using HTML, CSS, JavaScript, and Bootstrap 5. Pages include Home, Our Beans, Price List, Brewing Tips, Making Of (mock-ups & justifications), and Our Story (this reflection). The home page features a hero carousel and a registration form that validates user input and stores the first name in localStorage to personalize sub-pages. Bootstrap components—accordion, table, cards, and responsive grid—were used to create consistent UI patterns. All styles are centralized in an external stylesheet, and JavaScript is modularized in `scripts.js` for form validation and simple state persistence.

Challenges faced & how they were overcome: One challenge was ensuring the register form provided clear, actionable feedback while remaining accessible. To solve this, we used an alert container to list validation errors clearly and a success message area that acknowledges a valid submission. Another challenge was balancing aesthetics with maintainability: applying too many bespoke styles would have harmed maintainability. We resolved this by using Bootstrap as the structural foundation and keeping custom styles small and variables-based (colors and typography) so theme updates are straightforward. Lastly, ensuring responsive behavior for image-heavy sections required testing on different device widths and changing float behaviors via CSS media queries to avoid cluttered layouts on small screens.

What we learned technically: The project reinforced how Bootstrap can speed development while maintaining accessibility and responsiveness. We learned to combine Bootstrap utilities with a small, well-structured custom stylesheet to produce a unique brand identity without sacrificing maintainability. JavaScript lessons included robust client-side validation patterns and safe usage of `localStorage` to persist simple user state across pages. We also practiced writing semantic HTML, which improved document structure and made the site friendlier for screen readers.

Group work lessons: Coordinating assets and naming conventions early in the project saved time—agreeing on file names (e.g., `home.html`) and folder structure prevented merge conflicts. Clear division of tasks (design mock-ups, HTML templates, CSS theme, JavaScript behavior) worked well. Frequent small commits and simple documentation in a shared README would be recommended for future group work to improve traceability.

Improvements & future work: If we extend the site, we’d add server-side handling for form submissions and a small CMS for product management. Adding unit tests for JavaScript validation and implementing ARIA enhancements for further accessibility would raise quality. For a live e-commerce version, integrate secure checkout and inventory tracking.

Reflection summary: This project provided practical experience building a professional, responsive website from concept to implementation. The balance between design (colors, type, imagery) and technical decisions (semantic markup, centralized styles, modular JS) is central to maintainable web development. The outcome is a functional, attractive site that meets the assignment requirements and is ready for further enhancement.
